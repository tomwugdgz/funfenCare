## 📖 产品简介

**funfenCare** 是一款面向**独居老人及独居人士**的**安全监护 + 亲情连接**综合服务平台。通过**智能硬件监控**、**上门打卡服务**、**定制问候聊天**三大核心能力，为远在外地、国外的子女家人提供实时了解老人生活状况的安心方案。

### 核心价值主张

- **对老人**：安全有保障、日常有关怀、孤独有陪伴
- **对子女**：实时掌握父母状况、异常情况及时知晓、远程也能尽孝心
- **对社会**：降低独居老人意外风险、构建社区关爱网络

### 🎯 目标用户

| 用户角色 | 特征描述 | 核心诉求 |
|---------|---------|---------|
| 👴 独居老人 | 60岁以上，子女不在身边，独自生活 | 安全保障、日常问候、紧急求助 |
| 👩‍💼 独居人士 | 单身独居，有安全监护需求 | 安全打卡、异常报警、紧急联系 |
| 👨‍👩‍👧 子女家人 | 在外地/国外工作生活，关心父母状况 | 实时查看父母状态、接收异常通知、远程下单服务 |
| 🚪 服务人员 | 社区网格员、志愿者、职业陪护员 | 接单上门、打卡签到、记录反馈 |

## 🎬 使用场景

### 场景一：海外子女的安心守护

> 👩 女儿在美国 | 👴 父亲在广州独居

女儿通过 funfenCare App，每天查看父亲的 **WiFi体感活动报告**，确认父亲正常起居。每周一、三、五预约**上门打卡服务**，社区网格员上门敲门确认老人安全，拍照打卡后女儿即时收到通知。父亲生日当天，女儿在App内订购**定制问候聊天服务**，陪护员上门陪父亲聊天1小时，女儿通过App实时查看服务过程。

### 场景二：异常行为智能预警

> 👴 独居老人 | 📱 WiFi体感监控

WiFi体感设备检测到老人**超过24小时无活动轨迹**，系统自动触发一级预警，推送通知给子女和紧急联系人。子女确认后，立即通过App下单**紧急上门查看服务**，服务人员15分钟内到达现场，确认老人安全后反馈结果。

### 场景三：日常关怀与社交

> 👴 独居老人 | 👩‍💼 陪护员

老人通过App预约每日**"早安问候"电话**，陪聊员按时致电，询问健康状况、聊天解闷。老人还可以加入**兴趣社群**（如书法、戏曲、养生），与其他老人语音交流，减少孤独感。

## 🏗️ 五端架构

funfenCare 采用**五端分离架构**，通过AI分析实现**一对一个性化陪护**，解决传统摄像头侵犯隐私、手表佩戴不适的核心痛点。

### 架构概览

| 端 | 用户角色 | 核心功能 | 技术形态 |
|----|---------|---------|---------|
| **端1** | 独居用户 | 一键SOS、日常打卡、接受陪护 | App适老化版 / 微信小程序 |
| **端2** | 监护人（子女） | 实时监控、下单服务、健康报告 | App标准版 / 微信小程序 |
| **端3** | 后台管理员 | 用户管理、订单调度、数据分析 | Web管理后台 |
| **端4** | 全球AI监控 | 脱敏数据AI分析、模式识别 | Web4.0 MCP AI面板 |
| **端5** | 上门服务人员 | 接单、上门打卡、反馈记录 | 接单群 / 服务人员App |

### 核心设计理念

- ✅ **无摄像头**：WiFi体感替代摄像头，不拍摄画面，保护隐私
- ✅ **无穿戴设备**：无需老人佩戴手表/手环，消除抵触心理
- ✅ **AI个性化**：每个用户独立建模，AI学习生活习惯，提供专属陪护方案
- ✅ **五端协同**：独居用户、监护人、后台服务、全球AI监控、上门服务五端联动

## 📡 硬件接入（ruview）

本项目接入 [ruview](https://github.com/codemkt/ruview) 开源硬件方案，实现WiFi体感人类活动检测。

### ruview 简介

ruview 是基于 WiFi CSI（Channel State Information）的开源人体活动检测项目。通过分析WiFi信号的微小变化，可以在**无需摄像头、无需穿戴设备**的情况下，检测到房间内是否有人活动、活动轨迹和频率。

### 硬件清单

| 组件 | 型号推荐 | 数量 | 预估成本 |
|------|---------|------|---------|
| ESP32开发板 | ESP32-WROOM-32 | 2（发射+接收） | ¥50 |
| 天线 | 2.4GHz 全向天线 | 2 | ¥20 |
| 外壳 | 3D打印/亚克力 | 2 | ¥30 |
| 电源 | 5V/2A USB供电 | 2 | ¥20 |

### 安装步骤

1. **烧录固件**
   ```bash
   git clone https://github.com/codemkt/ruview.git
   cd ruview
   pio run --target upload --upload-port /dev/ttyUSB0
   ```

2. **部署设备**：将发射器和接收器分别放置在房间对角线位置

3. **配置数据上报**
   ```ini
   [server]
   url = https://api.funfencare.com/api/v1/hardware/csi
   token = your_device_token
   ```

4. **绑定设备到用户**：在 funfenCare App 内扫描设备二维码

> 💡 **隐私保护**：WiFi体感仅检测活动频率和轨迹，**不采集图像、不录音、不识别身份**

## 📦 安装与部署

### 环境要求

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Python | 3.9 | 3.11+ |
| Node.js | 16 | 18 LTS |
| PostgreSQL | 12 | 15+ |
| Redis | 6.0 | 7.0+ |
| InfluxDB | 2.0 | 2.7+ |
| EMQX (MQTT) | 5.0 | 5.4+ |

### 快速开始

1. **克隆仓库**
   ```bash
   git clone https://github.com/tomwugdgz/funfenCare.git
   cd funfenCare
   ```

2. **配置环境变量**
   ```bash
   cp .env.example .env
   # 编辑 .env 文件，填写必要配置
   ```

3. **安装后端依赖**
   ```bash
   cd backend
   pip install -r requirements.txt
   ```

4. **初始化数据库**
   ```bash
   createdb funfenCare
   python manage.py migrate
   ```

5. **启动服务**
   ```bash
   # 后端
   python manage.py runserver
   
   # 前端
   cd frontend && npm install && npm run dev
   
   # IoT网关
   docker run -d --name emqx -p 1883:1883 -p 8084:8084 emqx/emqx:5.4
   ```

### Docker Compose 一键部署

```bash
docker-compose up -d
docker-compose ps
docker-compose logs -f backend
```

## 💡 使用示例

### 👨‍👩‍👧 子女端操作

**绑定父母设备**
1. 打开 funfenCare App，进入"亲情圈"
2. 点击"添加成员"，输入父母手机号
3. 父母确认加入后，扫描家中WiFi体感设备二维码
4. 绑定成功，可实时查看父母活动状态

**预约上门打卡服务**
1. 进入"服务"页面，选择"上门打卡"
2. 选择父母地址，设置服务频率（如每周一三五上午9点）
3. 确认订单并支付（支持包月/包年套餐）
4. 服务人员接单后，按时上门敲门确认
5. 打卡完成后，子女即时收到通知和现场照片

### 👴 老人端操作

**一键SOS求助**
1. 打开App首页，点击红色大按钮"SOS"
2. 长按3秒确认（防误触）
3. 系统自动：拨打子女电话、发送当前位置、通知社区服务中心
4. 等待救援，保持通话

### 🚪 服务人员端操作

**接单上门打卡**
1. 收到新订单通知
2. 点击"接单"，查看老人地址和备注信息
3. 到达现场后，点击"开始打卡"
4. 敲门确认老人安全，拍摄现场照片
5. 填写服务记录（老人状态、异常情况备注）
6. 提交打卡记录，系统自动通知子女

## 🔒 安全与隐私

### 数据保护

- 数据传输全程加密（HTTPS/TLS 1.3）
- 敏感数据（手机号、身份证号、家庭地址）数据库字段加密
- 密码采用 bcrypt 哈希存储
- JWT Token有效期限制 + 刷新机制

### 隐私保护

- **WiFi体感不采集图像/音频**，仅检测活动频率和轨迹
- 上门服务照片仅用于打卡记录，服务完成后24小时自动删除
- 用户可随时导出个人数据或申请账号注销
- 隐私协议突出展示，首次使用时强制阅读确认

### 服务安全

- 服务人员实名认证 + 背景调查 + 定期培训
- 上门服务全程GPS轨迹记录
- 异常行为检测（频繁取消订单、超时服务等）
- 用户评价 + 投诉机制，24小时内处理反馈

4. 拿政府合同 → 复制到其他街道
关键动作：

对接3-5个养老机构SaaS集成商
