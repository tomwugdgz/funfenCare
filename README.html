📖 产品简介

funfenCare是一款面向独居老人及独居人士的安全监护+亲情连接综合服务平台。通过智能硬件监控、上门打卡服务、定制问候聊天三大核心能力，为远在外地、国外的子女家人提供实时了解老人生活状况的安心方案。

核心价值主张：
  • 对老人：安全有保障、日常有关怀、孤独有陪伴
  • 对子女：实时掌握父母状况、异常情况及时知晓、远程也能尽孝心
  • 对社会：降低独居老人意外风险、构建社区关爱网络

🎯 目标用户

用户角色 特征描述 核心诉求
👴 独居老人 60岁以上，子女不在身边，独自生活 安全保障、日常问候、紧急求助
👩‍💼 独居人士 单身独居，有安全监护需求 安全打卡、异常报警、紧急联系
👨‍👩‍👧 子女家人 在外地/国外工作生活，关心父母状况 实时查看父母状态、接收异常通知、远程下单服务
🚪 服务人员 社区网格员、志愿者、职业陪护员 接单上门、打卡签到、记录反馈

🎬 使用场景

场景一：海外子女的安心守护

👩 女儿在美国 👴 父亲在广州独居

女儿通过funfenCareApp，每天查看父亲的WiFi体感活动报告，确认父亲正常起居。每周一、三、五预约上门打卡服务，社区网格员上门敲门确认老人安全，拍照打卡后女儿即时收到通知。父亲生日当天，女儿在App内订购定制问候聊天服务，陪护员上门陪父亲聊天1小时，女儿通过App实时查看服务过程。

场景二：异常行为智能预警

👴 独居老人 📱 WiFi体感监控

WiFi体感设备检测到老人超过24小时无活动轨迹，系统自动触发一级预警，推送通知给子女和紧急联系人。子女确认后，立即通过App下单紧急上门查看服务，服务人员15分钟内到达现场，确认老人安全后反馈结果。

场景三：日常关怀与社交

👴 独居老人 👩‍💼 陪护员

老人通过App预约每日"早安问候"电话，陪聊员按时致电，询问健康状况、聊天解闷。老人还可以加入兴趣社群（如书法、戏曲、养生），与其他老人语音交流，减少孤独感。

✨ 核心功能

🛡️ 安全监护

📡

WiFi体感监控

通过WiFi信号波动检测人体活动，无需穿戴设备，保护隐私的同时实时监控起居规律。支持异常行为预警（长时间无活动、跌倒检测等）。

🚪

上门打卡服务

类似滴滴模式，子女下单预约，服务人员（网格员/志愿者/陪护员）上门敲门确认老人安全，GPS定位+拍照打卡，实时同步给子女。

🚨

紧急求助

一键SOS呼叫，直连子女+紧急联系人+社区服务中心。支持语音呼救、自动发送位置、自动拨打120。

💬 亲情连接

📞

定制问候聊天

预约陪聊员上门或电话聊天服务，可按话题定制（如"聊聊养生""听听戏曲""回忆往事"）。支持AI辅助陪聊+真人陪护双模式。

👨‍👩‍👧

亲情圈

子女、亲属、邻居、社区网格员组成"亲情圈"，共同关注老人状态。支持多端实时同步、异常通知、服务协同。

📊

健康日报

自动生成老人每日活动报告（起居规律、活动量、情绪状态），推送给子女。支持周报/月报趋势分析。

📱 适老化设计

  • 大字模式：全局字体放大，高对比度配色
  • 语音辅助：语音播报操作指引、语音输入
  • 极简交互：核心功能3步内完成，底部仅4个Tab（首页、监控、服务、我的）
  • 防误触：大按钮、长按确认、操作回退

📡 硬件接入（ruview）

本项目接入 ruview 开源硬件方案，实现WiFi体感人类活动检测。

ruview 简介

ruview 是基于 WiFi CSI（Channel State Information）的开源人体活动检测项目。通过分析WiFi信号的微小变化，可以在无需摄像头、无需穿戴设备的情况下，检测到房间内是否有人活动、活动轨迹和频率。

硬件清单

组件 型号推荐 数量 预估成本
ESP32开发板 ESP32-WROOM-32 2（发射+接收） ¥50
天线 2.4GHz 全向天线 2 ¥20
外壳 3D打印/亚克力 2 ¥30
电源 5V/2A USB供电 2 ¥20

安装步骤

  1. 烧录固件 
    # 克隆 ruview 仓库
git clone https://github.com/codemkt/ruview.git
cd ruview

# 使用 PlatformIO 编译烧录
pio run --target upload --upload-port /dev/ttyUSB0
  2. 部署设备

    将发射器和接收器分别放置在房间对角线位置，确保信号覆盖主要活动区域（客厅、卧室）。

  3. 配置数据上报 
    # 在 ruview 配置文件中设置funfenCare平台地址
# config.ini
[server]
url = https://api.fufen-companion.com/api/v1/hardware/csi
token = your_device_token
  4. 绑定设备到用户

    在funfenCareApp内扫描设备二维码，完成设备与老人的绑定。

隐私保护：WiFi体感仅检测活动频率和轨迹，不采集图像、不录音、不识别身份，严格遵守隐私保护原则。

📦 安装与部署

环境要求

组件 最低版本 推荐版本
Python 3.9 3.11+
Node.js 16 18 LTS
PostgreSQL 12 15+
Redis 6.0 7.0+
InfluxDB 2.0 2.7+
EMQX (MQTT) 5.0 5.4+

快速开始

  1. 克隆仓库 
    git clone https://github.com/your-org/fufen-companion.git
cd fufen-companion
  2. 配置环境变量 
    cp .env.example .env
# 编辑 .env 文件，填写必要配置
    必要配置项：
    • DATABASE_URL - PostgreSQL连接字符串
    • REDIS_URL - Redis连接地址
    • INFLUXDB_URL - InfluxDB连接地址
    • MQTT_BROKER_URL - MQTT Broker地址
    • JWT_SECRET - JWT签名密钥
    • WECHAT_APP_ID - 微信小程序AppID
  3. 安装后端依赖 
    cd backend
pip install -r requirements.txt
  4. 初始化数据库 
    # 创建数据库
createdb fufen_companion

# 运行迁移
python manage.py migrate
  5. 启动后端服务 
    # 开发模式
python manage.py runserver

# 生产模式（gunicorn）
gunicorn manage:app -w 4 -b 0.0.0.0:8000
  6. 启动前端 
    cd frontend
npm install
npm run dev
  7. 启动IoT网关 
    # 安装 EMQX
docker run -d --name emqx -p 1883:1883 -p 8084:8084 emqx/emqx:5.4

Docker Compose 一键部署

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f backend

# 停止服务
docker-compose down
生产环境注意：请勿使用默认配置直接部署到生产环境。务必修改密钥、启用HTTPS、配置防火墙规则、设置数据库备份策略。

🔒 安全与隐私

数据保护

  • 数据传输全程加密（HTTPS/TLS 1.3）
  • 敏感数据（手机号、身份证号、家庭地址）数据库字段加密
  • 密码采用 bcrypt 哈希存储
  • JWT Token有效期限制 + 刷新机制

隐私保护

  • WiFi体感不采集图像/音频，仅检测活动频率和轨迹
  • 上门服务照片仅用于打卡记录，服务完成后24小时自动删除
  • 用户可随时导出个人数据或申请账号注销
  • 隐私协议突出展示，首次使用时强制阅读确认

服务安全

  • 服务人员实名认证 + 背景调查 + 定期培训
  • 上门服务全程GPS轨迹记录
  • 异常行为检测（频繁取消订单、超时服务等）
  • 用户评价 + 投诉机制，24小时内处理反馈
适老化安全特性：针对老年用户群体，特别加强防诈骗提醒、异常交易拦截、亲属辅助验证等功能。
